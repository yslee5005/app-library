## D001: Anonymous-First 인증 (2026-04-09)

**결정:** 모든 앱에서 로그인 화면을 제거하고, 앱 시작 시 Supabase `signInAnonymously()`를 자동 호출한다.

**이유:**
- 시니어 유저(50-70대)가 로그인에서 60-70% 이탈
- Apple/Google 구독은 앱 계정 없이 구매 가능 (RevenueCat anonymous ID)
- Supabase 익명 계정은 RLS가 정상 동작 (`auth.uid()` 발급됨)
- 나중에 `linkIdentity()`로 데이터 병합 가능

**대안 고려:**
- ❌ 소셜 로그인 필수 → 이탈률 높음
- ❌ 기기 ID 기반 → Apple 정책 위반, 앱 삭제 시 유실
- ❌ 로컬 저장만 → 기기 변경 시 데이터 유실

**적용:** 모든 앱 (`CLAUDE.md` Always 규칙)

## D002: 기도 Dashboard vs QT Dashboard 분리 (2026-04-09)

**결정:** 기도 후와 QT 묵상 후에 완전히 다른 Dashboard를 보여준다.

**이유:**
- 기도 = 나 → 하나님 (내가 말씀드림) → 기도 요약, 말씀 매칭, 기도 제목 추적
- QT = 하나님 → 나 (말씀을 통해 들음) → 묵상 분석, 적용 코칭, 관련 지식
- 목회자/신학자 관점에서 두 활동의 목적과 기대 결과가 다름

**적용:** Abba (`currentPrayerModeProvider`로 분기)

## D003: 인스타그램 스타일 커뮤니티 (2026-04-09)

**결정:** 커뮤니티 피드를 카드 기반에서 인스타그램 스타일 (풀 너비, 댓글 바텀시트, 더블탭 좋아요)로 변경.

**이유:**
- 카드 + 접기 방식의 댓글이 불안정 (overflow, state 관리 복잡)
- 인스타그램 UX는 시니어도 익숙함
- 무한 스크롤이 성능상 유리 (20개씩 cursor pagination)

**적용:** Abba 커뮤니티, `packages/feed/`로 표준화

## D004: AI 프롬프트에 문체 지시 (2026-04-09)

**결정:** OpenAI 프롬프트에 "단편 소설가처럼 쓰라"는 문체 지시를 추가.

**이유:**
- 기본 AI 출력이 "보고서체" — 사건 나열, 반복 문장 구조
- 감각 묘사, 내면 독백, 비유/은유를 지시하면 품질이 극적으로 향상
- 시니어 유저에게 "읽는 즐거움"이 리텐션에 직결

**구체적 지시:**
- 감각 묘사 (시각, 청각, 촉각)
- 내면 독백 포함
- 문장 구조 다양화 (같은 패턴 반복 금지)
- 비유/은유 최소 2개
- "보여주기, 말하지 않기"

**적용:** Abba `openai_service.dart` 기도/QT 프롬프트

## D005: Lilsquare 디자인 테스트 라우트 (2026-04-09)

**결정:** 외부 사이트(Lilsquare.com) 디자인을 `/lilsquare` 별도 라우트에서 테스트한 후, 마음에 드는 부분만 본 사이트에 병합.

**이유:**
- 기존 사이트를 직접 수정하면 롤백이 어려움
- 나란히 비교 가능 (`/` vs `/lilsquare`)
- 팀/클라이언트에게 A/B 보여주기 용이

**적용:** Blacklabelled 웹 (`/lilsquare/*` 12개 페이지)

## D006: 하네스 엔지니어링 적용 (2026-04-09)

**결정:** OpenAI 하네스 엔지니어링 5원칙을 App Library에 적용.

**원칙 매핑:**
1. 에이전트가 볼 수 없는 것은 없다 → `ARCHITECTURE.md`, `DECISIONS.md`
2. 실패 원인은 환경 부족 → 반응형 테스트, l10n 검증, 레이어 린터
3. 문서보다 기계적 강제 → pre-commit hooks, CI 스크립트
4. 에이전트에 시각 → 테스트 결과 자동 피드백
5. 설명서가 아닌 지도 → ARCHITECTURE.md (50줄 조감도)

**이유:**
- Ralph(Claude)에게 좋은 프롬프트를 쓰는 것보다, Ralph가 잘 작동하는 환경을 만드는 것이 더 효과적
- PROMPT 작성 시간 50% 감소 (컨텍스트가 repo에 있으므로)
- overflow/l10n/RLS 버그 90% 사전 방지

**적용:** 전체 repo
