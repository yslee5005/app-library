# Claude Code의 메모리 및 컨텍스트 시스템 상세 정리

## 개요
Claude Code는 프로젝트별, 사용자별 맞춤형 지시사항을 제공하기 위해 **4단계 계층형 메모리 구조**를 지원합니다.

## 4단계 메모리 계층도

### 1단계: 관리형 메모리 (최저 우선순위)
- **경로**: `/etc/claude-code/CLAUDE.md` (또는 조직 설정 경로)
- **특징**: 시스템 관리자가 설정한 조직 전체 지시사항
- **범위**: 머신의 모든 사용자에게 적용
- **정책**: 사용자/프로젝트 파일로 재정의 불가능

### 2단계: 사용자 메모리
- **경로**: `~/.claude/CLAUDE.md` 및 `~/.claude/rules/*.md`
- **용도**: 개인 전역 설정 (모든 프로젝트에 적용)
- **예시**: 코딩 스타일, 기본 언어, Git 사용자명
- **특징**: 저장소에 커밋되지 않음 (개인용)

### 3단계: 프로젝트 메모리
- **경로들**:
  - `CLAUDE.md`
  - `.claude/CLAUDE.md`
  - `.claude/rules/*.md`
- **검색 방식**: 현재 작업 디렉토리부터 루트까지 상위 디렉토리 탐색
- **용도**: 팀 전체 공유 지시사항 (버전 관리 포함)
- **활용**: 아키텍처 노트, 테스트 명령어, 프로젝트 규칙

### 4단계: 로컬 메모리 (최고 우선순위)
- **경로**: `CLAUDE.local.md` (상위 디렉토리 모두 검색)
- **특징**: `.gitignore`에 포함되어야 함
- **용도**: 개인 워크플로우 설정, 지역 환경 변수, 미공개 단축키

**핵심 원칙**: 현재 작업 디렉토리에 가까울수록 우선순위가 높음

## 파일 발견 알고리즘

```
실행 순서:
1. 관리형 메모리 로드
2. 사용자 메모리 로드
3. 프로젝트/로컬 메모리 (루트 → CWD 순서로)
```

- 구현: `utils/claudemd.ts`의 `getMemoryFiles()` 함수
- **메모이제이션**: 세션 중 발견된 파일 목록 캐시
- 리로드: `/memory` 명령어 또는 세션 재시작

## `@include` 디렉티브

다른 파일을 참조하여 컨텍스트에 포함시킵니다.

### 구문 지원 형식

| 구문 | 해석 |
|------|------|
| `@filename` | 포함 파일 디렉토리 기준 상대 경로 |
| `@./relative/path` | 명시적 상대 경로 |
| `@~/home/path` | 홈 디렉토리 기준 경로 |
| `@/absolute/path` | 절대 경로 |

### 규칙
- **코드 블록 내 무시**: 펜스 코드 블록과 인라인 코드 내 경로는 처리 안 함
- **순환 참조 차단**: 자동 감지 및 스킵
- **누락된 파일**: 자동으로 무시
- **최대 깊이**: 5단계
- **지원 파일 형식**: 텍스트 기반만 (`.md`, `.ts`, `.py`, `.json` 등)
- **보안**: 프로젝트 외부 경로는 명시적 승인 필요

### 예제

```markdown
# 프로젝트 CLAUDE.md

@./docs/architecture.md
@./docs/conventions/typescript.md

커밋 전 `bun test` 실행 필수
```

## `.claude/rules/*.md` - 세분화된 규칙 파일

단일 CLAUDE.md 대신 여러 파일로 분산 관리 가능:

```
프로젝트/
├── CLAUDE.md
└── .claude/
    └── rules/
        ├── testing.md
        ├── typescript-style.md
        └── git-workflow.md
```

### 경로 범위 지정 (Frontmatter)

```markdown
---
paths:
  - "src/api/**"
  - "src/services/**"
---

의존성 주입 항상 사용. 구현체 직접 임포트 금지.
```

- **Glob 패턴 매칭**: 지정된 경로 내 파일만 규칙 적용
- **효과**: 대규모 프로젝트에서 컨텍스트 최소화

## 파일 크기 제한

- **권장 최대**: 40,000자 (`MAX_MEMORY_CHARACTER_COUNT`)
- **초과 시**: 경고 표시, 전체 내용 미읽음 가능성
- **전략**: 집중도 높고 간결한 메모리 파일 유지

## 메모리가 동작에 미치는 영향

어셈블된 메모리 블록 접두사:

> "지침은 기본 동작을 **재정의**하며 정확히 따라야 함"

**결과**: 프로젝트 규칙 강제, 특정 작업 제한, 도메인 지식 주입 가능

## 각 단계별 사용 시기

| 메모리 | 용도 |
|--------|------|
| **관리형** | 조직 정책, 보안 규칙, 배포 설정 |
| **사용자** | 응답 언어, 커밋 스타일, 개인 단축키 |
| **프로젝트** | 테스트 명령어, 아키텍처, 네이밍 규칙, 내부 문서 링크 |
| **로컬** | 환경 경로, 개인 디버깅 노트, 팀 설정 오버라이드 |

## 실제 CLAUDE.md 예제

```markdown
# 내 프로젝트

## 명령어

- 빌드: `bun run build`
- 테스트: `bun test`
- 린트: `bun run lint`
- 타입 체크: `bun run typecheck`

## 아키텍처

모노레포. 핵심 로직은 `packages/core`. CLI 진입점은 `src/index.ts`.

## 규칙

- `zod/v4` 스키마 검증 필수
- `any` 금지, `unknown` + 타입 가드 사용
- 비동기 함수는 명시적 에러 처리 필수
- 테스트는 `__tests__/` 디렉토리 (같은 수준)

## Git

- 브랜치: `feat/<ticket>-설명`
- 커밋 전 `bun test && bun run typecheck` 실행
- `console.log` 포함 금지

@./docs/api-conventions.md
```

## `/memory` 명령어

REPL 내 `/memory` 입력 시:
- 현재 로드된 메모리 파일 목록 표시
- 메모리 파일 직접 편집
- 저장 시 컨텍스트 자동 리로드

**팁**: "CLAUDE.md에 2칸 들여쓰기 규칙 추가해줘" 같이 요청 가능 -- Claude가 적절한 파일을 찾아 작성

## 메모리 로딩 비활성화

| 방법 | 효과 |
|------|------|
| `CLAUDE_CODE_DISABLE_CLAUDE_MDS=1` | 모든 메모리 파일 로딩 비활성화 |
| `--bare` 플래그 | 자동 발견 스킵, `--add-dir`만 사용 |
| `claudeMdExcludes` 설정 | Glob 패턴으로 제외할 파일 지정 |
