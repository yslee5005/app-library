# Claude Code 작동 방식 - 상세 추출

## 개요
Claude Code는 터미널 기반 코딩 에이전트로, "연속적인 에이전트 루프"를 실행합니다. 사용자 요청을 읽고, 추론하며, 도구를 호출하고, 결과를 관찰한 후 반복합니다.

## 에이전트 루프 구조

### 6단계 순환 프로세스

**1단계: 사용자 메시지 전송**
- 터미널 대화형 모드 또는 `--print`/stdin을 통한 비대화형 모드
- 메시지가 대화 기록에 추가됨

**2단계: 컨텍스트 조립**
- `context.ts`에서 시스템 프롬프트 구성
- 포함 요소: 현재 날짜, git 상태, CLAUDE.md 메모리 파일, 사용 가능한 도구 목록
- 대화당 한 번 구성되고 메모이제이션됨

**3단계: Claude의 추론 및 도구 선택**
- Anthropic API로 대화 전송
- 모델이 하나 이상의 `tool_use` 블록 방출

**4단계: 권한 확인**
- 각 도구 호출 실행 전 현재 권한 모드 평가
- 결과에 따라 자동 승인, 확인 요청, 또는 차단

**5단계: 도구 실행 및 결과 반환**
- 승인된 도구 호출 실행
- 결과(파일 내용, 명령 출력 등)를 `tool_result` 블록으로 추가

**6단계: 루프 지속**
- 모델이 추가 도구를 호출하거나 최종 텍스트 응답 생성
- 도구 호출이 남지 않을 때까지 반복

### 중요 특징
"루프는 전적으로 터미널 프로세스에서 실행되며, 원격 실행 서버가 없습니다."

## 컨텍스트 로딩

### 시스템 컨텍스트 (`getSystemContext()`)
- 현재 브랜치 및 기본/메인 브랜치
- Git 사용자명
- `git status --short` 출력(2,000자 이상 시 잘림)
- 최근 5개 커밋(`git log --oneline`)
- 캐시 무효화 주입 스트링(디버깅용)

조건: `CLAUDE_CODE_REMOTE=1` 설정 시 또는 설정에서 git 지시사항 비활성화 시 스킵

### 사용자 컨텍스트 (`getUserContext()`)
- CLAUDE.md 메모리: 4단계 계층 구조로 발견
  - 관리 → 사용자 → 프로젝트 → 로컬
  - `CLAUDE_CODE_DISABLE_CLAUDE_MDS=1`로 비활성화 가능
- 현재 날짜: "Today's date is YYYY-MM-DD" 형식

### 메모이제이션
두 컨텍스트 블록 모두 `lodash/memoize`를 사용하여 대화 기간 동안 메모이제이션됨. `setSystemPromptInjection()`으로 캐시 즉시 초기화.

## 도구 실행 모델

### 권한 확인 결과 테이블

| 권한 결과 | 동작 |
|---------|------|
| `allow` | 즉시 실행, 결과 대화에 추가 |
| `ask` | 확인 대화 렌더링 및 일시 중지 |
| `deny` | 호출 거부, 오류 결과 반환 |

### 권한 모드 동작
- **bypassPermissions**: 모든 확인 스킵
- **acceptEdits**: 파일 편집 도구 자동 승인, bash 명령은 확인 요청
- 안전한 읽기 전용 도구(Read, Glob, Grep): 모든 모드에서 자동 승인

## 대화형 vs 비대화형 모드

### 대화형(REPL) 모드
- 기본 경험
- React/Ink를 사용한 라이브 터미널 UI
- 스트리밍 출력, 도구 확인, 스피너 애니메이션 표시

### 비대화형/인쇄 모드
- `--print` 또는 stdin 파이핑으로 활성화
- UI 렌더링 없음
- stdout으로 출력(스크립트/CI 파이프라인 활용)

### 서브-에이전트 (Task 도구)
- `Task` 도구(`AgentTool`)를 통해 에이전트 생성 가능
- 독립된 대화 및 선택적 제한된 도구 세트로 자체 루프 실행
- 로컬 또는 원격 실행 가능
- 완료 시 부모 에이전트에 결과 반환

## 대화 저장 및 재개

### 저장 방식
- JSON 트랜스크립트 파일로 디스크 저장(`~/.claude/` 기본값)
- 고유 세션 ID 부여

### 재개 방법
- `--resume <session-id>`로 특정 대화 재개
- `--resume` 단독 명령으로 목록에서 선택

### 재개 시 동작
- 전체 메시지 기록 로드
- 메모리 파일 재발견(처음과 다를 수 있음)
- 권한 모드 기본값으로 리셋(세션에 저장되지 않은 경우)

### 대화 컴팩션
장시간 대화는 주기적으로 컴팩션됨(최하위 메시지 요약). 전체 원본 트랜스크립트는 디스크에 보존되며, 컴팩션은 API 전송 내용만 영향.

## 쿼리 엔진

각 루프 "턴"은 `query.ts` 호출로 구동됨.

### 주요 기능
- 실시간 터미널로 토큰 출력 스트리밍
- `tool_use` 블록을 적절한 도구 핸들러로 디스패치
- 턴당 토큰 및 도구 호출 예산 강제
- 도구 결과 수집 및 다음 모델 호출 전 추가
- 컨텍스트 윈도우 채워질 때 컴팩션 트리거

### 결과 크기 관리
- 각 도구는 `maxResultSizeChars` 속성 보유
- 결과가 한계 초과 시 임시 파일로 저장
- 모델은 파일 경로가 포함된 미리보기 수신(컨텍스트 오버플로우 방지)
