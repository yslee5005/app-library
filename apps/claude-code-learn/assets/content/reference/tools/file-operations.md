# 파일 작업 도구 (File Operations)

## 개요

Claude Code는 로컬 파일시스템을 다루기 위해 세 가지 전용 도구를 제공합니다: Read(읽기), Edit(편집), Write(작성).

---

## Read 도구

### 기능

파일의 내용을 읽어 줄 번호와 함께 반환합니다.

### 매개변수

- **file_path** (필수, 문자열): 절대 경로 필요 (상대 경로 불가)
- **offset** (숫자): 읽기 시작 줄 번호 (1부터 시작)
- **limit** (숫자): 반환할 최대 줄 수 (기본값 2,000줄)

### 출력 형식

`cat -n` 스타일 포맷팅으로 각 줄에 1부터 시작하는 줄 번호 붙음:

```
   1  import { z } from 'zod'
   2
   3  export const schema = z.object({
   4    name: z.string(),
   5  })
```

### 지원 파일 유형

| 유형 | 동작 |
|------|------|
| 텍스트 파일 | 줄 번호와 함께 원본 텍스트 반환 |
| 이미지 (PNG, JPG) | 시각적으로 표시됨 (멀티모달) |
| PDF | 10페이지 초과 시 `pages` 매개변수 사용 (예: "1-5"), 최대 20페이지 |
| Jupyter 노트북 (.ipynb) | 모든 셀, 출력, 코드, 텍스트, 시각화 반환 |

### 주의사항

- Read는 디렉토리를 나열할 수 없음
- 디렉토리 검사는 Bash 도구의 `ls` 사용
- 파일 미변경 시 캐시된 스텁 반환 (불필요한 재읽기 방지)

## Edit 도구

### 기능

파일 내에서 정확한 문자열 교체 수행

### 매개변수

- **file_path** (필수): 수정할 파일의 절대 경로
- **old_string** (필수): 찾을 정확한 문자열 (공백, 들여쓰기 포함)
- **new_string** (필수): 대체 문자열 (빈 문자열로 삭제 가능)
- **replace_all** (불린): true 설정 시 모든 출현 부분 교체

### 작동 방식

정규식 해석 없음 - 순수 문자열 비교로 매칭. `old_string`이 여러 번 나타나면 `replace_all: true` 필수.

### 필수 요건

Edit 호출 전에 먼저 해당 파일을 Read로 읽어야 함.

### 들여쓰기 매칭

Read 출력에서 줄 번호 접두사 뒤의 내용을 사용합니다.

---

## Write 도구

### 기능

새 파일 생성 또는 기존 파일 완전 덮어쓰기

### 매개변수

- **file_path** (필수): 절대 경로
- **content** (필수): 작성할 완전한 내용

### 사용 기준

**Edit 사용:**
- 기존 파일의 하나 이상 섹션 수정
- 변수/기호 이름 변경
- 코드 블록 추가/제거

**Write 사용:**
- 새 파일 처음부터 생성
- 파일 완전히 재작성
- 기존 내용과 완전히 다른 새 콘텐츠

### 주의사항

- 기존 파일 덮어쓰기 시 먼저 Read 호출
- 문서 파일(*.md, README.*) 자동 생성 안 함
