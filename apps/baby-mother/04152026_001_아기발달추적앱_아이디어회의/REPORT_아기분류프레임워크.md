# Deep Analysis: 아기를 어떻게 분류해야 하는가?

> **질문**: "카테고리를 애기를 어느식으로 나눠야해?"
>
> **결론**: **4계층(Layer) 점진적 분류** — 온보딩은 2-3개 질문으로 경량화하고, 시간이 지나면서 자동으로 정교해지는 구조가 최적.

---

## 1. 변수별 변이 설명력 순위

연구 데이터 기반으로 "어떤 변수가 변이를 가장 많이 줄이는가"를 정리하면:

```
변이 설명력 순위 (높음 → 낮음):

1위: 월령 (Age)           ████████████████████ (R² 34-43%)
2위: 출생특성 (GA+BW)     ██████████████       (R² 17-30%)
3위: 수유방식 (Feeding)    ██████████           (중간 효과, 6개월 미만에서 최강)
4위: 수면환경 (Sleep)      ████████             (R² ~20%, 수면 영역 한정)
5위: 기질 (Temperament)    ██████               (r = .34-.49, 수면 문제 예측)
6위: 성별 (Sex)            ████                 (단독 효과 작음, 상호작용에서 의미)
```

### 핵심 규칙

| 규칙 | 수치 근거 |
|------|----------|
| **변수 2-3개가 최적** | 4개 이상 = diminishing returns + 그룹당 n 급감 |
| **그룹당 최소 25-30명** | Oakes 2017 시뮬레이션: 24명 미만은 위양성/위음성 높음 |
| **변수 추가당 ΔR² ~ 3-5%p** | 상호작용 효과 존재하지만 한계적 |
| **월령이 다른 변수 효과를 조절** | 수유방식 효과는 6개월 이후 약화 |

## 2. 기존 앱들의 분류 방식 비교

| 앱 | 핵심 분류 변수 | 온보딩 질문 수 | 전략 |
|----|--------------|-------------|------|
| **Wonder Weeks** | 출산예정일 1개만 | **1개** | 극단적 미니멀 → Leap 타임라인 즉시 제공 |
| **CDC Milestone** | 생년월일 + 성별 + 조산여부 | **5개** | 의학적 정확성 중시 |
| **Glow Baby** | 이름 + 생년월일 + 출생여부 | **3개** | 빠른 시작 → 추후 점진 수집 |
| **Huckleberry** | 생년월일 + 예정일 + 수면패턴 + 수유 + 일정 | **8-10개** | 깊은 개인화 → 즉시 스케줄 생성 |

### UX 연구 데이터

| 지표 | 수치 |
|------|------|
| 앱 첫 3일 이탈률 | **77%** |
| 질문 8개→3개 축소 시 이탈률 | **30% 감소** |
| 5개 초과 시 참여도 | **15% 하락** |
| 개인화 온보딩 시 리텐션 | **+40%** |

> **교훈**: 질문이 많으면 즉시 제공하는 가치도 커야 한다. Huckleberry는 10개 질문 대신 "즉시 일일 스케줄 생성"이라는 높은 가치를 제공해서 성립.

## 3. 제안: 4계층 점진적 분류 프레임워크

```
┌─────────────────────────────────────────────┐
│  Layer 1: 즉시 분류 (온보딩 2-3문항)          │
│  → 월령 + 조산여부                           │
│  → 변이 37-65% 축소                          │
│  → 0초 → 즉시 기본 범위 제공                  │
├─────────────────────────────────────────────┤
│  Layer 2: 환경 분류 (Day 1-3, 점진 수집)      │
│  → 수유방식 + 수면환경                        │
│  → 추가 변이 41-70% 축소                      │
│  → "더 정확한 범위를 보여드릴까요?" 유도       │
├─────────────────────────────────────────────┤
│  Layer 3: 기질 분류 (Day 3-7, 선택)           │
│  → 8문항 간이 스크리닝 또는 자동 추론          │
│  → Easy / Difficult / Slow-to-warm           │
│  → 수면 문제 예측력 r=.34-.49                 │
├─────────────────────────────────────────────┤
│  Layer 4: 개인 기준선 (Day 7+, 자동)          │
│  → 5일+ 데이터 → 개인 패턴 클러스터링         │
│  → 유사 그룹 자동 매칭                        │
│  → "우리 아이 타입" 확정                      │
└─────────────────────────────────────────────┘
```

---

### Layer 1: 즉시 분류 (온보딩)

**수집 정보**: 2-3개 질문

| 질문 | 이유 | 분류 효과 |
|------|------|----------|
| **생년월일 (또는 출산예정일)** | 변이 설명력 1위 (R² 34-43%), 모든 앱 공통 필수 | 월령 기반 5-6개 구간으로 분류 |
| **조산 여부 (예정일과 출생일 차이)** | 교정월령 계산 필수, CDC 앱도 필수 수집 | 조산아는 별도 기준 적용 |
| **성별** (선택) | WHO 성장차트 성별 분리, 단 수면/배변에는 효과 작음 | 성장 모니터링에만 사용 |

**즉시 제공 가치**: 교정월령 기반 발달 범위 + 수면/수유 참고 범위

**월령 구간 제안** (연구 근거 기반):

| 구간 | 근거 |
|------|------|
| **0-6주** (신생아기) | 패턴 미확립, 매우 높은 변이 |
| **6주-3개월** (초기) | 패턴 형성 시작, 수유방식 효과 최대 |
| **3-6개월** (중기) | 수면 패턴 급변, 야간 통잠 시작 |
| **6-12개월** (후기) | 변이 안정화, 야간수면 590분 고정 |
| **12-18개월** (걸음마 초기) | 수면 통합, 낮잠 1회로 전환 |
| **18-24개월** (걸음마 후기) | 패턴 거의 확립 |

---

### Layer 2: 환경 분류 (Day 1-3)

**수집 시점**: 앱 사용 시작 후 1-3일 내, 자연스러운 맥락에서 수집

**수집 방법**: 첫 기록 시 "더 정확한 비교를 위해 알려주세요" 형태

| 질문 | 분류 | 변이 축소 효과 |
|------|------|-------------|
| **수유 방식** | 완전 모유 / 혼합 / 완전 분유 | 배변 변이 **41-70% 축소**, 수면 35분 차이 |
| **수면 환경** | 공동수면 / 같은방-다른침대 / 독립방 | 야간각성 CV **57% 축소** |

**왜 온보딩이 아니라 Day 1-3인가?**
- 온보딩 질문 5개 초과 시 참여도 15% 하락
- 첫 기록 시 맥락적으로 물으면 자연스러움
  - 수유 기록 시: "어떤 방식으로 수유하시나요?"
  - 수면 기록 시: "아이가 어디서 자나요?"

**이 시점에서의 분류 조합**:

```
월령 6구간 × 수유 3종 × 수면환경 3종 = 54개 하위그룹 (이론적 최대)
→ 현실적: 월령 6구간 × 수유 3종 = 18개 그룹 (수면은 수면 지표에만 적용)
```

### Layer 3: 기질 분류 (Day 3-7)

**수집 시점**: 앱 사용 3-7일 후, "아이를 더 잘 이해하기 위한 퀴즈" 형태로 제공

**두 가지 방법** (택 1 또는 병행):

#### 방법 A: 8문항 간이 스크리닝

Thomas & Chess + Rothbart 결합, 부모 친화적 질문:

| # | 질문 | 측정 차원 | 척도 |
|---|------|----------|------|
| 1 | 아이의 수면/수유 시간이 매일 비슷한가요? | **규칙성** (Rhythmicity) | 1(매우 불규칙)~5(규칙적) |
| 2 | 새로운 음식/장소/사람 처음 접할 때 반응은? | **접근/회피** | 1(강한 거부)~5(적극 수용) |
| 3 | 수유/수면 루틴이 바뀌면 얼마나 빨리 적응? | **적응성** | 1(매우 느림)~5(즉시) |
| 4 | 배고프거나 불편할 때 울음 강도는? | **반응 강도** | 1(약함)~5(매우 강렬) |
| 5 | 평소 아이의 기분은? | **기분** | 1(자주 울적)~5(밝음) |
| 6 | 아이가 울 때 달래기 쉬운가요? | **조절 능력** | 1(어려움)~5(쉬움) |
| 7 | 낮에 팔다리 움직임이 얼마나 활발? | **활동/외향성** | 1(차분)~5(활발) |
| 8 | 한 물건에 집중하는 시간은? | **주의 지속** | 1(짧음)~5(오래) |

**분류 결과**:

| 유형 | 비율 | 핵심 패턴 특성 |
|------|------|-------------|
| **편안한 아이** (Easy) | ~40% | 규칙적 수면/수유, 빠른 적응, 달래기 쉬움 |
| **활발한 아이** (Difficult) | ~10% | 불규칙적, 강한 반응, 느린 적응, 야간각성 40%↑ |
| **신중한 아이** (Slow-to-warm) | ~15% | 초기 위축, 낮은 활동, 점진적 적응 |
| **혼합형** | ~35% | 위 유형의 조합 |

> **주의**: "Difficult"라는 단어는 부모에게 부정적 → **"활발한 아이"**, **"에너지가 넘치는 아이"** 등으로 리프레이밍 필요

#### 방법 B: 데이터 기반 자동 추론

5-7일간의 기록 데이터에서 **규칙성(Rhythmicity)을 자동 계산**:

```
규칙성 점수 = 1 - (수면시작시간 SD / 평균수면시작시간)

높은 규칙성 (SD < 30분): "규칙적 타입"
중간 규칙성 (SD 30-60분): "유연한 타입"
낮은 규칙성 (SD > 60분): "자유로운 타입"
```

- 배변, 수유 시간에도 같은 계산 적용
- 부모에게 질문하지 않고도 기질 경향을 추론 가능
- 단, 정확도는 방법 A보다 낮음

### Layer 4: 개인 기준선 (Day 7+, 완전 자동)

**수집 방법**: 기록 데이터가 5일 이상 축적되면 자동 작동

**작동 원리** (Huckleberry SweetSpot 참고):

```
Step 1: 개인 데이터에서 패턴 추출
        → 평균 수면시간, 수면시작시간, 야간각성 횟수 등

Step 2: 같은 Layer 1-3 카테고리 내 유사 사용자와 비교
        → "4개월 / 모유수유 / 편안한 타입" 그룹 중 유사 패턴 매칭

Step 3: 개인 기준선(baseline) 확정
        → "우리 아이의 정상 범위: X~Y"

Step 4: 이탈 감지
        → 기준선에서 벗어나면 "평소와 다릅니다" 알림
```

**이 시점에서 사용자가 보는 것**:

```
┌──────────────────────────────────┐
│  우리 아이 프로필                  │
│                                  │
│  월령: 4개월 (교정 3.5개월)        │
│  타입: 모유수유 · 같은방 수면       │
│  기질: 편안한 아이 (규칙적 타입)    │
│                                  │
│  ─── 우리 아이의 범위 ───          │
│  수면: 14.2~15.8시간/일           │
│  (같은 타입 평균: 14.8시간)        │
│  수유: 7~9회/일                   │
│  배변: 2~4회/일                   │
│                                  │
│  오늘: 수면 15.1시간 ✅ 정상 범위   │
└──────────────────────────────────┘
```

## 4. 기질별 수면/수유/배변 차이 수치

기질 분류가 실제로 의미 있는 차이를 만드는지 확인:

### 수면 패턴 차이

| 지표 | Easy Baby | Difficult Baby | 차이 |
|------|-----------|---------------|------|
| 입면 시간 | 평균 수준 | **+49% 길음** (r=.49, p<.01) | 유의 |
| 야간 각성 | 적음, 스스로 재입면 | **+40% 많음** (r=.40, p<.05) | 유의 |
| 야간 깨어있는 시간 | 짧음 | **+35% 길음** (r=.35, p<.05) | 유의 |
| 수면 문제 정도 | 낮음 | **+34% 높음** (r=.34, p<.05) | 유의 |

### 수유 패턴 차이

| 지표 | Easy Baby | Difficult Baby |
|------|-----------|---------------|
| 수유 규칙성 | 규칙적 (매일 비슷한 시간) | 불규칙적 (예측 불가) |
| 새 음식 반응 | 빠른 수용 | 강한 거부 → 반복 노출 필요 |
| 모유수유 지속 | 지속 가능성 높음 | 조기 중단 위험 증가 |

### 배변 패턴 차이

| 지표 | Easy Baby | Difficult Baby |
|------|-----------|---------------|
| 배변 규칙성 | 매일 비슷한 시간 | 예측 불가능 |
| 배변 시 반응 | 차분 | 강한 울음/보채기 |

> 출처: Reeb-Sutherland et al. (PMC4417414), Schwichtenberg & Goodlin-Jones (PMC3190304)

## 6. 왜 이 순서인가? (연구 근거 요약)

| 순서 | 변수 | 왜 이 순서? |
|------|------|-----------|
| **1순위: 월령** | 모든 발달/행동 변수에서 R² 34-43%로 압도적. 월령이 다른 변수 효과를 조절(moderate)함 |
| **2순위: 수유방식** | 배변 변이 41-70% 축소, 수면에 35분 차이. 단, 6개월 이후 효과 감소 |
| **3순위: 기질** | 수면 문제 예측 r=.34-.49. 수유/배변 규칙성에 직접 영향. 단, 측정에 시간 필요 |
| **4순위: 개인 데이터** | 최종적으로 가장 정확하지만 데이터 축적 시간 필요 (최소 5일) |
| **수면환경은?** | 수면 지표에만 영향 (R²~20%). 별도 분류축이 아닌 "수면 범위 계산 시 보정 변수"로 사용 |
| **성별은?** | 발달 마일스톤 106개 중 102개(96%)가 성별 동일. 성장차트에만 분리 적용 |

## 7. 기존 앱과의 차별화

| | Wonder Weeks | Huckleberry | CDC Milestone | **우리 앱 (제안)** |
|---|:-----------:|:-----------:|:------------:|:----------------:|
| 분류 변수 수 | 1개 (예정일) | 2-3개 (월령+수면) | 3개 (월령+성별+조산) | **4계층 점진적** |
| 기질 반영 | ❌ | ❌ | ❌ | **✅ (8문항 또는 자동)** |
| 수유방식 분류 | ❌ | 부분적 | ❌ | **✅ (배변/수유 범위 분리)** |
| 개인 기준선 | ❌ | ✅ (수면만) | ❌ | **✅ (수면+수유+배변)** |

### 핵심 차별화 한 줄 요약

## 토론 포인트

1. **기질 분류의 라벨링**: "Difficult baby"를 부모에게 어떻게 표현할 것인가? → "에너지가 넘치는 아이", "감수성이 풍부한 아이" 등 리프레이밍 필요
2. **Layer 2 수집 타이밍**: 첫 기록 시 물을 것인가, 별도 팝업으로 물을 것인가?
3. **혼합수유의 경계**: 모유:분유 비율에 따라 패턴이 다른데, 어떻게 구분할 것인가? (예: "주로 모유" / "반반" / "주로 분유")
4. **데이터 부족 초기**: 앱 런칭 초기에 자체 데이터가 없을 때 → 학술 데이터 기반 범위 사용 → 사용자 증가 후 자체 클러스터링으로 전환
5. **기질은 변한다**: 영아 기질은 완전히 고정이 아님 (Bornstein 2019). 3-6개월 주기로 재분류 필요한가?
