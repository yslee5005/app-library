# Deep Analysis: 아기가 진짜 원하는 것

> **질문**: "report를 기반으로, 정말 아기가 원하는 것 — 태아, 신생아, 영아의 관점에서 — 은 무엇인가?"
>
> **결론**: 기존 보고서 4개는 모두 **부모/관찰자/앱 관점**으로 설계되었다. "아기가 실제로 원하는 것"에 대한 과학적 근거를 종합하면, 아기의 욕구는 **5계층 위계(hierarchy)**로 정리되며, 기존 추적 앱들은 가장 하위 계층(생리적 기록)에만 집중하고 있다. 더 나아가, 추적 행위 자체가 아기에게 해로울 수 있다(Technoference). **"최고의 아기 앱은 부모가 앱을 덜 보게 만드는 앱"**이다.

---

## 0. 이 보고서의 위치: 기존 4개 보고서의 상위 프레임워크

```
                    ┌─────────────────────────┐
                    │  이 보고서 (WHY)          │
                    │  "아기가 원하는 것"        │
                    │  = 앱의 존재 이유          │
                    └────────────┬────────────┘
                                 │
         ┌──────────┬────────────┼────────────┬──────────┐
         ▼          ▼            ▼            ▼          ▼
    정확도/신뢰  카테고리화   분류프레임워크  점진적      (향후)
    딜레마       로 변이축소  4-Layer        프로파일링   MVP 설계
    (WHAT NOT)   (HOW)       (HOW)          (WHEN)      (WHAT)
```

기존 보고서들은 **어떻게(HOW)** 만들 것인가에 집중했다. 이 보고서는 **왜(WHY)** 만드는가를 정의한다.

## 1. Baby's Needs Hierarchy: 아기 욕구의 5계층

연구 근거를 종합하면, 아기의 욕구는 다음과 같은 위계를 형성한다:

```
                    ┌───────────────────┐
                    │ Level 5           │
                    │ 예측 가능한 세계    │ ← 아기가 만드는 리듬
                    │ Predictable World │
                    ├───────────────────┤
                    │ Level 4           │
                    │ 개별적 이해        │ ← "나만의 패턴을 알아줘"
                    │ Individual        │
                    │ Recognition       │
                    ├───────────────────┤
                    │ Level 3           │
                    │ 공동조절           │ ← Serve & Return, 터치
                    │ Co-regulation     │
                    ├───────────────────┤
                    │ Level 2           │
                    │ 감각적 연속성      │ ← 자궁→세상 전환
                    │ Sensory           │   ★ 어떤 앱도 안 다룸
                    │ Continuity        │
                    ├───────────────────┤
                    │ Level 1           │
                    │ 생리적 항상성      │ ← 수유, 수면, 체온
                    │ Physiological     │   (기존 앱들이 다 함)
                    │ Homeostasis       │
                    └───────────────────┘
```

### 각 Level과 기존 앱의 대응 현황

| Level | 아기의 욕구 | 기존 앱 대응 | 빈 공간(Blue Ocean) |
|-------|-----------|------------|-------------------|
| 1 | 배고프지 않기, 적정 체온, 충분한 수면 | ✅ 기록/추적 (Huckleberry, Glow 등) | 거의 없음 (레드오션) |
| 2 | 자궁 환경의 연속성 (소리, 냄새, 접촉) | ❌ **전혀 없음** | **완전한 블루오션** |
| 3 | 내 신호에 맞춘 즉각적 반응 | ⚠️ 울음 분석 앱 (과학적 한계) | 반응 능력 "교육" 도구 |
| 4 | 나만의 고유한 패턴 인정 | ⚠️ 백분위 비교만 (개인화 부족) | 개인 기준선 접근법 |
| 5 | 예측 가능한 리듬 (스케줄이 아님) | ⚠️ 스케줄 강제 (부모 주도) | 아기 주도 리듬 추적 |

## 2. 발달 단계별 상세: 아기가 원하는 것의 과학적 근거

### 2-1. 태아 (Fetus): "나는 이미 배우고 있어"

태아는 수동적 존재가 아니다. 자궁 내에서 이미 **학습, 기억, 선호**를 형성한다.

#### 미각/후각 학습

| 연구 | 발견 | 의미 |
|------|------|------|
| **Ustun et al. 2022** (Psychological Science) | 당근 맛 노출 → 태아 "웃는 얼굴(lip-corner puller)" 증가 / 케일 맛 노출 → "우는 얼굴(cry-face gestalt)" 증가 | **자궁 내에서 이미 맛 선호가 존재** |
| **2025 Appetite** | 자궁 내 경험한 냄새에 신생아도 긍정 반응(울음↓, 웃음↑) 유지 | **태아기 기억이 출생 후까지 지속** |

> 태아는 엄마가 먹는 음식의 맛을 양수를 통해 경험하며, 이 경험은 출생 후 음식 선호에까지 영향을 미친다. 태아에게 "익숙한 맛"은 안전 신호다.

#### 청각 학습

| 연구 | 발견 | 의미 |
|------|------|------|
| **체계적 리뷰 (PMC 2023)** | 25-28주부터 내/외부 소리 구별 가능 | 엄마 목소리 = 최초의 "안전 소리" |
| **Stanford 2025** | 미숙아에게 엄마 목소리 녹음 → 언어 경로 성숙도 향상 | 목소리가 뇌 구조를 물리적으로 바꿈 |
| **태아 심박 연구** | 엄마 목소리에 태아 심박수 안정화 | 25주부터 "인식 + 학습" 진행 중 |

#### 습관화와 조건화

| 능력 | 시기 | 의미 |
|------|------|------|
| **습관화(Habituation)** | 30-38주 | 반복 자극에 대한 반응 감소 = 기억 형성 |
| **고전적 조건화** | 32주~ | 예측적 반응 가능 = "다음에 뭐가 올지 안다" |
| **프라이밍** | 3rd trimester | 모체 생리적 변화 → 태아 심박 반응 = **엄마 상태를 읽고 있음** |

#### 태아가 원하는 것 요약

```
1. 감각적 일관성 — 익숙한 소리(엄마 목소리, 심장박동), 익숙한 맛(엄마 식단)
2. 예측 가능성 — 규칙적 패턴에 대한 학습과 적응
3. 모체 안정성 — 엄마의 스트레스가 cortisol 경로로 태아에게 전달됨
```

> **앱 시사점**: 임신 중 "엄마의 행동이 아기에게 어떤 영향을 주는지" 과학적 근거 기반 콘텐츠. 단순 "~하세요" 조언이 아닌, "왜 그런지" 설명. 기존 REPORT_임신부터시작하는점진적프로파일링.md의 Phase 0.5 콘텐츠와 연결.

### 2-2. 신생아 (0-28일): "나는 아직 적응 중이야"

#### 4th Trimester — 가장 과소평가된 개념

인간 신생아의 뇌는 성인의 **25%**에 불과하다. 대부분의 포유류는 60-90% 완성 상태로 태어난다.

| 비교 | 출생 시 뇌 완성도 | 의미 |
|------|-----------------|------|
| 말(horse) | ~90% | 태어나자마자 걷고 달림 |
| 침팬지 | ~40% | 수개월간 엄마에게 매달림 |
| **인간** | **~25%** | **가장 미성숙한 상태로 출생** |

> 이것은 인간의 **직립보행** 때문이다. 골반이 좁아져 뇌가 더 커지기 전에 태어나야 한다. 진화적으로 인간 아기는 **12주 더 자궁에 있어야 했던** 존재다.

#### 일주기 리듬(Circadian Rhythm) 발달 타임라인

| 시기 | 상태 | 부모가 알아야 할 것 |
|------|------|------------------|
| **0-2주** | 낮밤 구분 거의 없음 | **정상이다.** 아기가 "밤낮이 바뀐" 게 아니라, 아직 리듬이 없는 것 |
| **8-9주** | 호르몬이 일주기 따르기 시작 | 최초의 패턴이 나타나기 시작 |
| **12-16주** | 야간 수면 패턴 확립 | 비로소 "밤에 더 자는" 패턴 형성 |
| **6-18개월** | 일주기 리듬 완전 고정 | 안정적인 수면-각성 사이클 |

**핵심**: 생후 첫 몇 주는 **"sensitive window of circadian programming"** — 빛 노출이 일주기 프로그래밍에 결정적 영향.

> **앱 시사점**: "수면 스케줄을 만들어라"가 아니라 "아기의 리듬이 **발달하는 과정**을 이해하라". 0-2주에 "아기가 밤낮이 바뀌었어요" 라는 부모의 불안에 "아직 리듬이 없는 게 정상입니다" 라고 알려주는 것.

#### 신생아가 원하는 것 요약

```
1. 자궁과 비슷한 환경 — 따뜻함, 감싸짐, 엄마 심장박동, 엄마 냄새
2. 나만의 적응 속도 — 스케줄 강요 아닌 점진적 전환
3. 빛 환경의 자연스러운 전환 — 낮에 빛, 밤에 어둠 (일주기 프로그래밍)
4. 지속적 신체 접촉 — 캥거루 케어, 안아주기
```

### 2-3. 영아 (1-12개월): "내 신호에 반응해줘"

#### Serve & Return — 뇌를 만드는 상호작용

Harvard Center on the Developing Child의 핵심 개념:

```
아기가 옹알이, 표정, 울음으로 "서브(serve)"를 보냄
       │
       ▼
부모가 눈맞춤, 말, 안아주기로 "리턴(return)"
       │
       ▼
신경 연결이 강화됨 (매초 100만+ 시냅스)
       │
       ▼
이것이 반복되면 → 건강한 뇌 구조(brain architecture) 형성
       │
       ▼
반복되지 않으면 → 독성 스트레스 반응 활성화 → 뇌 발달 저해
```

#### Still Face Experiment — 아기는 반응을 "기대"한다

Ed Tronick(1975~)의 실험:

| 단계 | 상황 | 아기 반응 |
|------|------|---------|
| 1. 정상 상호작용 | 엄마가 미소, 말, 반응 | 아기도 미소, 옹알이, 활발 |
| 2. Still Face | 엄마가 갑자기 무표정 | **2초 만에** 고통 반응 시작 |
| 3. 재시도 | 아기가 엄마 관심 끌기 시도 | 미소, 가리키기, 울음 |
| 4. 포기 | 반응 없음 지속 | 고개 돌림, 자기위안 행동, 울음 |

> **이 실험이 증명한 것**: 아기는 이미 "사회적 상호작용의 규칙"을 이해하고 있다. "내가 신호를 보내면, 반응이 와야 한다"는 기대가 이미 형성되어 있다.

> **앱과의 연결**: 부모가 앱에 기록하느라 아기를 안 보는 순간 = 아기에게 **미니 "Still Face"** 경험. Technoference의 과학적 근거.

#### 옥시토신 회로 — 터치가 뇌를 만든다

| 연구 | 발견 |
|------|------|
| **Feldman Lab (2017)** | 옥시토신 + 도파민의 선조체 통합이 애착을 점화(ignite). 영유아기 옥시토신 수용체 밀도가 **2-3세에 정점** |
| **FAU 캥거루 케어 연구** | 피부접촉 → 좌전두엽 활성화 (고차 인지/감정조절 영역) |
| **Vittner et al. (2018)** | 캥거루 케어 → 옥시토신 중간~큰 효과 증가 + 코르티솔 반응성 중간 효과 감소 |
| **Stanford 2024** | 미숙아: 캥거루 케어 받은 그룹이 **1세 발달지연 비율 유의미하게 낮음** |
| **Micheletti 2025 (Infancy)** | **안아주기(holding)**가 음성(vocalizing)보다 더 강력한 미주신경(vagal) 조절 효과 |
| **Eroglu 2026 (Int J Dev Neurosci)** | 캥거루 케어 = 쉽게 적용 가능하고 비용 효과적인 방법, 영아-아동 건강에 다각적 이점 |

> **터치는 단순한 '좋은 것'이 아니라 뇌 구조를 만드는 물리적 힘이다.**

#### 공동조절(Co-regulation)의 문화적 차이

| 문화 | 주요 공동조절 방식 | 연구 근거 |
|------|-----------------|---------|
| 서구 (유럽계) | **Face-to-face** 상호작용 | 표정, 시선, 음성 |
| 아시아, 아프리카, 태평양 | **Touch** 중심 | 접촉, 안아주기, 업어주기 |

> **한국 시장 시사점**: 한국 문화에서 "업기", "포대기", "공동수면" 등 접촉 중심 양육이 전통적으로 강함. 이것이 과학적으로도 옥시토신 경로를 통한 공동조절의 강력한 방법임을 인정하고, 앱이 이를 지지(validate)하는 콘텐츠 필요.

#### 울음: 보편적 유형은 없다

| 주장 | 과학적 근거 |
|------|-----------|
| **Dunstan Baby Language**: 5가지 보편적 울음 (neh=배고픔, owh=졸림 등) | **학술적 검증 실패** — 인간 영아는 보편적 이산 울음 레퍼토리를 진화시키지 않았다 |
| **울음 분석 앱**: 90-95% 정확도 | 과의존 시 **부모 직관 발달 저해** — "앱이 아니라 부모가 아기를 알아야 한다" |
| **2023 Nature Communications Psychology** | 울음 = "일반적 고통의 **등급적(graded)** 표현" + **개별 서명(individual signature)** |

> **핵심**: 울음에는 "배고픔 울음", "잠 울음" 같은 보편 카테고리가 없다. 대신 각 아기만의 고유한 패턴이 있고, 부모가 그 패턴을 시간에 걸쳐 배우는 것이 정상적 과정.

> **앱 시사점**: "이 울음은 배고픔 80%입니다" 같은 판단을 내리면 안 됨. 대신 "이 아기의 울음 패턴이 지난 3일과 어떻게 다른지" 보여주고, 부모가 스스로 연결점을 찾도록 도움.

#### 영아가 원하는 것 요약

```
1. 내 신호에 맞춘 즉각적 반응 (Serve & Return)
2. 충분한 신체 접촉 (옥시토신 경로)
3. 나만의 의사소통 패턴을 이해해주는 것 (보편적 해석이 아닌 개별 학습)
4. 예측 가능한 리듬 (스케줄이 아닌, 아기 주도의 반복적 패턴)
5. 부모가 나를 보고 있다는 확신 (Technoference 없는 상호작용)
```

## 3. 추적 앱의 근본적 역설: Technoference

### 3-1. Technoference란?

**Technology + Interference** = 기술이 부모-아이 상호작용을 방해하는 현상

| 연구 | 발견 |
|------|------|
| **Frontiers in Developmental Psychology 2025** | 객관적 측정된 스마트폰 사용 → 모-영아 상호작용 질 **저하** |
| **PMC 2022** | 스마트 모니터링이 "불안 해소" 주장하지만 **객관적 확인 연구 없음** |
| **PMC 2022** | 사용자 **~20%**가 추적 도구가 "스트레스/불안 유발"한다고 응답 |
| **PMC 2019** | 상용 육아 앱 중 **임상 시험에서 테스트된 앱 = 0개** |

### 3-2. 추적 앱의 역설 구조

```
부모의 의도:      "아기를 더 잘 돌보고 싶어" (선한 동기)
       │
       ▼
앱의 약속:        "모든 걸 기록하면 패턴이 보여요"
       │
       ▼
실제 행동:        수유 후 즉시 앱 열어서 시간/양 기록
       │
       ▼
아기의 경험:      수유 후 엄마가 나를 안 보고 화면을 봄 (Still Face)
       │
       ▼
과학적 결과:      Serve & Return 단절 → 상호작용 질 저하
       │
       ▼
장기적 영향:      앱이 아기를 위한 것이었는데, 아기에게 해로운 결과
```

### 3-3. 추가 역설들

| 역설 | 설명 |
|------|------|
| **불안 역설** | 기록할수록 "정상 범위"를 벗어날까 불안 → 더 많이 기록 → 더 많은 불안 |
| **비교 역설** | "우리 아기는 평균 대비..." → 불필요한 비교 → 아기의 개별성 무시 |
| **판단 역설** | 앱이 "수면 부족" 경고 → 부모가 아기에게 수면 스케줄 강요 → 아기 스트레스 |
| **대체 역설** | 앱에 의존할수록 → 부모의 직관적 아기 읽기 능력 발달 저해 |

### 3-4. 하지만 앱이 해야 할 일은 있다

Technoference가 문제라고 해서 앱이 무용하다는 것은 아니다. 문제는 **어떤 앱인가**.

| 해로운 앱 | 유익한 앱 |
|----------|---------|
| "매번 기록하세요" → 빈번한 스크린 타임 | "하루 한 번만 확인하세요" → 최소 스크린 타임 |
| "정상 범위 밖입니다" → 불안 유발 | "이런 변화는 자연스럽습니다" → 안심 제공 |
| "이 울음은 배고픔입니다" → 판단 대행 | "이 패턴은 지난주와 비슷하네요" → 관찰 보조 |
| 데이터가 목적 | 부모-아기 상호작용의 질이 목적 |

## 4. 아기 관점의 앱 설계 원칙

### 4-1. 핵심 전환: Tracking App → Understanding App

| 차원 | Baby Tracking App (기존) | Baby Understanding App (새로운) |
|------|:----------------------:|:----------------------------:|
| **철학** | "아기의 모든 것을 기록" | "아기의 신호를 읽는 능력 키우기" |
| **기록 빈도** | 가능한 한 많이 | **최소한만** (Technoference 방지) |
| **수유** | 시간/양/횟수 기록 | "아기가 배고파하는 신호" 교육 |
| **수면** | 시간 기록 → 스케줄 제안 | "아기의 리듬이 발달하는 과정" 이해 |
| **성장** | 백분위 비교 차트 | "내 아기만의 성장 궤적" 추적 |
| **울음** | AI 분석/분류 | "이 아기의 의사소통 패턴" 함께 배우기 |
| **핵심 KPI** | 기록 횟수, DAU | **부모-아기 상호작용 시간** |

### 4-2. 5 Design Principles (아기 관점)

#### Principle 1: "내 리듬을 존중해줘" (Respect My Rhythm)

- 스케줄 제안 금지 — 아기가 만드는 리듬을 따라가기
- 일주기 리듬 발달 단계를 부모에게 교육
- "우리 아기는 아직 12주 전이라 밤낮 구분이 없는 게 정상이에요"

**연구 근거**:
- 부모 주도 루틴 → 모유수유 기간 단축 + 비만 위험↑ (PMC 2014)
- 반응적 양육 → 수면 지속↑, 야간 각성↓, BMI↓ (INSIGHT study)
- 단, 일관된 취침 "루틴"(순서)은 감정/행동 조절에 긍정적 (Penn State 2025)

> **루틴(routine) ≠ 스케줄(schedule)**. 루틴은 "목욕→마사지→수유→자장가" 같은 **순서의 예측 가능성**. 스케줄은 "8시에 자야 해" 같은 **시간의 강제**.

#### Principle 2: "나에게 반응해줘" (Respond to Me)

- 앱 사용 시간 최소화 → 아기와의 상호작용 시간 극대화
- "Serve & Return" 개념을 자연스럽게 교육
- 패시브 트래킹 (자동 기록) 선호, 수동 입력 최소화

#### Principle 3: "자궁에서의 기억을 이어줘" (Continue My Womb Experience)

- 4th Trimester 전환 가이드 제공 (어떤 앱도 안 다루는 블루오션)
- "엄마 심장박동", "백색소음", "피부접촉" 등의 과학적 근거 제공
- 임신 중 태동 데이터 → 출생 후 "이 아기가 자궁에서 좋아했던 것" 연결

#### Principle 4: "나를 일반화하지 마" (Know ME, Not Babies in General)

- 백분위 비교 최소화 → 개인 기준선 중심
- 보편적 울음 유형 분류 금지 → 이 아기만의 패턴 추적
- "평균 아기는 이래요" 대신 "우리 아기는 이래요"

**연구 근거**: 보편적 울음 유형은 존재하지 않음 (2023 Nature Communications Psychology)

#### Principle 5: "엄마가 나를 보고 있다는 걸 알고 싶어" (Be Present)

- Technoference 방지를 앱 설계의 핵심 원칙으로
- "기록보다 안아주기" 메시지
- 하루 한 번 회고 모드 (실시간 기록 지양)
- 터치/접촉 시간을 추적 항목으로 (역설적으로 스크린 시간↓ 유도)

## 5. 기존 보고서와의 통합 프레임워크

### 5-1. 기존 보고서 재해석 (아기 관점 렌즈 적용)

| 기존 보고서 | 부모/앱 관점 결론 | 아기 관점 재해석 |
|-----------|----------------|---------------|
| **정확도신뢰딜레마** | "판단하지 않으면 정확도 문제가 사라진다" | ✅ 맞음 — 아기도 판단받고 싶지 않다 |
| **카테고리화로 변이축소** | "카테고리화하면 범위 37-70% 축소" | ⚠️ 주의 — 아기는 "분류되지 않기"를 원함. 분류는 **도구**이지 **목적**이 아님 |
| **분류프레임워크** | "4-Layer 점진적 분류" | ✅ 점진적 접근은 맞음 — 아기도 점진적으로 자기를 드러냄 |
| **점진적프로파일링** | "임신부터 데이터 수집" | ⚠️ 주의 — 데이터 수집이 목적이 아니라, "아기 이해"가 목적. 수집은 수단 |

### 5-2. 통합된 앱 설계 방향

```
기존 보고서의 HOW (어떻게 만들까)
+ 이 보고서의 WHY (왜 만드는가)
= 통합 설계 원칙

┌─────────────────────────────────────────────┐
│                                             │
│  WHY: 아기와 부모의 상호작용 질을 높이기     │
│       (Baby Understanding)                  │
│                                             │
│  ┌─────────────────────────────────────┐    │
│  │ HOW:                                │    │
│  │  - 카테고리화로 범위 축소 (도구)      │    │
│  │  - 4-Layer 점진적 분류 (도구)        │    │
│  │  - 임신부터 점진적 데이터 수집 (도구)  │    │
│  │  - 판단 않고 기록 (원칙)             │    │
│  └─────────────────────────────────────┘    │
│                                             │
│  GUARD RAILS:                               │
│  - Technoference 방지 (최소 스크린 타임)     │
│  - 스케줄 강제 금지 (아기 주도 리듬)         │
│  - 보편적 판단 금지 (개별 패턴만)            │
│  - 비교 최소화 (개인 기준선 중심)            │
│                                             │
└─────────────────────────────────────────────┘
```

## 6. 앱이 "할 수 있는 것"과 "할 수 없는 것"

### 할 수 있는 것 (앱의 적절한 역할)

| Level | 앱이 할 수 있는 것 | 구현 방식 |
|-------|-----------------|---------|
| 1 | 기록의 편의성 제공 | 최소 입력, 패시브 트래킹 |
| 2 | 4th Trimester 교육 | 과학 기반 콘텐츠 (블루오션) |
| 3 | Serve & Return 교육 | "아기가 보내는 신호" 가이드 |
| 4 | 개인 패턴 시각화 | "우리 아기만의 리듬" 대시보드 |
| 5 | 리듬 발달 단계 안내 | "지금 이 시기에는 이런 변화가 정상" |

### 할 수 없는 것 (앱의 한계)

| 앱이 할 수 없는 것 | 이유 | 대안 |
|------------------|------|------|
| 아기를 안아주기 | 물리적 행동 | 안아주기의 중요성을 부모에게 알려주기 |
| 아기 울음 정확히 해석 | 보편적 울음 유형이 없음 | 부모가 패턴을 배우도록 보조 |
| 부모 대신 반응하기 | 부모가 직접 해야 함 | 반응 능력 교육 |
| 아기의 발달 속도 변경 | 생물학적 과정 | 발달 단계를 이해시키기 |

## 토론 포인트

1. **Technoference와 앱 비즈니스의 모순**: DAU(일일활성사용자)를 높이는 것이 비즈니스 목표인데, "앱을 덜 보게 만드는 앱"은 어떻게 수익화할 수 있는가? → 구독 모델? 세션 시간이 아닌 "이해도 향상"을 가치로?

2. **패시브 트래킹의 실현 가능성**: 웨어러블 없이 앱만으로 자동 기록이 가능한가? → 최소 입력 설계 vs 기기 연동 전략

3. **4th Trimester 콘텐츠**: 이것이 진짜 블루오션인가? 기존 임신/육아 앱들이 왜 안 다루고 있는가? → 콘텐츠의 깊이 vs 의학적 책임 문제?

4. **"판단하지 않는" 것의 한계**: 정말 위험한 신호(발달 지연, 학대 징후 등)는 판단해야 하지 않는가? → 레드플래그 시스템의 설계

5. **한국 문화적 맥락**: 포대기, 업기, 공동수면 등 한국 전통 양육이 과학적으로 옥시토신 경로를 활성화한다는 것을 앱이 "인정"하고 "지지"하면 차별화 가능

6. **이전 보고서의 "카테고리화"와 이 보고서의 "개별화"는 모순인가?**: 카테고리화는 "범위를 좁히기 위한 도구"이지 "아기를 규정하는 레이블"이 아님. 이 구분을 UX에서 어떻게 표현할 것인가?
