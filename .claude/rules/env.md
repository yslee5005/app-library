# Environment Variable Rules

## Required: flutter_dotenv Runtime Loading
- Do not use `String.fromEnvironment`
- Use `dotenv.load(fileName: '.env.runtime')`

## File Structure

```
app-library/
├── .env              ← single master (all shared keys + server-only), gitignored
├── .env.example      ← tracked template for .env
├── apps/<app>/
│   ├── .env.client   ← app-specific keys (bundle ids, per-platform RC keys, etc.)
│   ├── .env.example  ← tracked template for .env.client
│   └── .env.runtime  ← auto-generated by scripts/prepare_env.sh (gitignored)
└── scripts/
    └── prepare_env.sh
```

| File | Scope | Examples |
|------|-------|----------|
| `.env` | All apps, shared + server-only | SUPABASE_URL, SUPABASE_ANON_KEY, GEMINI_API_KEY, SENTRY_DSN, SUPABASE_SERVICE_ROLE_KEY |
| `.env.client` | One app | APP_ID, ENV, GOOGLE_IOS_CLIENT_ID, REVENUECAT_IOS_KEY, REVENUECAT_ANDROID_KEY |
| `.env.runtime` | One app (generated) | `.env` minus server-only keys + `.env.client` concatenated |

## Server-only Keys

Any secret that must stay off the client bundle goes into the `EXCLUDED_KEYS`
array inside `scripts/prepare_env.sh`. Current list:

- `SUPABASE_SERVICE_ROLE_KEY`

The script strips those keys while building `.env.runtime`, so `.env` can
stay as a single source of truth without leaking server credentials into
the Flutter bundle.

## Before Running an App

```bash
./scripts/prepare_env.sh abba            # → apps/abba/.env.runtime
./scripts/prepare_env.sh blacklabelled   # → apps/blacklabelled/.env.runtime
```

Re-run whenever `.env` or `apps/<app>/.env.client` changes.

## Access Pattern

```dart
import 'package:flutter_dotenv/flutter_dotenv.dart';

await dotenv.load(fileName: '.env.runtime');
final url = dotenv.env['SUPABASE_URL'] ?? '';
```
